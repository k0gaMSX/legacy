    • Why use a synthesizable core?

    A synthesizable core allows you to move your design from one process technology to another without redesign. You can switch operating voltages, change process geometries; and change CAD tools or standard cell libraries (low power or high performance) without redesigning the core.

    In June 1998, Mentor Graphics and Synopsys published the Reuse Methodology Manual (RMM) that describes design practices for creating reusable IP. The eZ80 softcore was independently verified to be RMM compliant.

    The eZ80 softcore is written in Verilog and followed a rigorous design methodology that created a fully synchronous design. The design methodology did not allow latches, feedback loops, or multi-cycle paths. Any logic synthesizer can generate RTL from the Verilog source.

    Using the eZ80 softcore, you can test your design using an FPGA from Xilinx or Altera. You can use the fabrication facility of your choice whether it is in house or an independent foundry.

    Why use an 8-bit core instead of a 16- or 32-bit core?

    The basic reasons why you should use an 8-bit core: time-to-market, cost, code density, and performance. Designing a device with an 8-bit core reduces your design cycle because an 8-bit bus is easy to understand and interface with off-the-shelf peripherals. One design house went from project start to GDSII tapeout in 11 weeks with no prior knowledge of the eZ80.

    Most embedded applications are sensitive to cost fluctuations of even a few pennies. The eZ80 softcore is the smallest core available in its performance range. The Z80 instruction set has been optimized over a 25-year period for embedded applications. Our enhanced Z80, the eZ80, uses this knowledge to create a core that has the greatest code density available. For example, opcodes can be from one to three bytes long. Frequently used opcodes require only one byte, thereby, reducing code size.

    Some 16- and 32-bit cores have on-chip caches which greatly increase core size. In addition, these cores typically have 16-bit address and data paths which greatly increase routing area and thus die size.

    By minimizing the core area and the memory area required, we have reduced your die size and total system on a chip cost.

    Another reason to use the eZ80 is performance. Many 16- and 32-bit cores use multiple clocks per instruction to execute. Even though they may execute at 20MHz, their effective MIP rating is much less. The eZ80 has a single-cycle fetch and many instructions can execute in a single cycle. Besides the eZ80 has been simulated running at 80MHz in 0.25m process, and working silicon has been reported running at 50MHz.

    Has the eZ80 been implemented in silicon?

    Yes. Prior to announcing the licensing program, ZiLOG created a general purpose platform board that allows a user to evaluate the capabilities and operation of the eZ80 core. The FPGA contains the eZ80 core and a serial port.

    ZiLOG developed its eZ80 Webserver part using the softcore. The eZ80 Webserver device is currently in production operating at 50MHz in 0.35m process technology and goes by the part number eZ80S190.

    For ZiLOG to design a device using it’s own softcore is easy. The real test is whether a licensee can do it. We are proud to say that one of our licensees’s created a design using the eZ80 and that this part is in production.

    Are tests and a testbench provided?

    The eZ80 comes with a test bench and tests to verify the operation of the core. The test bench and tests also covers the ZiLOG Debug Interface (ZDI) and the Multiply and Accumulate engine (MACC).

    In addition, we provide scripts for scan insertion if you wish to have scan chains in your final product.

    How difficult is it to connect peripherals to the eZ80?

    The eZ80 uses a standard bus with memory mapped I/O. Peripheral devices are treated just like another memory location. A peripheral device’s registers are addressed, read and written just like a byte of memory.

    What are the performance characteristics of the eZ80?

    Since the eZ80 is synthesizable, the best answer is it depends on your choice of process technology, operating voltage, synthesis constraints, standard cell library and CAD tools. We can provide information about our experience, but the best way is for you to synthesize the core in your desired technology and measure the results.
